# Priority Levels & Filtering

## Requirements

Users can assign a priority level — High, Medium, or Low — to each todo at creation time. If no priority is chosen, the todo defaults to Medium. After a todo is saved, the user can change its priority at any time.

Each todo item displays a clear visual indicator (such as a coloured badge or dot) that reflects its current priority level, making it easy to scan the list at a glance.

A filter bar at the top of the list lets the user narrow what is shown. Priority filters (All / High / Medium / Low) and status filters (All / Active / Completed) work independently of each other, so users can combine them — for example, showing only High-priority Active todos.

The list is automatically ordered so that incomplete todos appear sorted High → Medium → Low. Completed todos always appear below all incomplete todos, regardless of their priority.

## Acceptance Criteria

- A new todo can be saved with High, Medium, or Low priority; omitting a selection saves it as Medium
- Each todo item shows a visible priority indicator that matches its current priority level
- A todo's priority can be changed after creation, and the indicator and list position update immediately
- The filter bar provides All / High / Medium / Low options for priority and All / Active / Completed options for completion status
- Priority and status filters can be applied together or independently
- When a filter is active, only matching todos are visible; resetting to "All" restores the full list
- Incomplete todos are ordered High → Medium → Low within the list
- Completed todos appear below all incomplete todos regardless of their priority
